# Kore Data Schema & Markdown Format

The source of truth in Kore is the flat Markdown document. QMD relies completely on the structure of these documents for optimal retrieval and categorization.

This document defines the physical layout, file naming conventions, and Zod validation schemas for the generated `Memory` files.

---

## 1. Directory Structure

Kore organizes the raw `.md` files into coarse-grained buckets. These buckets map directly to QMD's "collections".

By default, files are placed under `~/kore-data/[type]/`. The type is derived from the structural nature of the data, NOT the semantic topic.

*   `~/kore-data/places/`: For geolocation-bound memories (Restaurants, cafes, venues).
*   `~/kore-data/media/`: For articles, videos, books, podcasts.
*   `~/kore-data/notes/`: For abstract thoughts, scratchpads, generic Apple Notes.
*   `~/kore-data/people/`: Interactions, preferences, contact data.

### 1.1 File Naming Convention
Names are generated by slugifying the primary entity, title, or topic extracted by the LLM.
*   **Format:** `lowercase_words_separated_by_underscores.md`
*   **Slugify algorithm:** Lowercase the input, replace whitespace and special characters with `_`, strip all non-alphanumeric characters except `_`, collapse consecutive underscores, and truncate to 50 characters.
*   **Collision:** If the destination file exists, a unique hash suffix (first 4 hex characters of a random UUID) is appended (e.g. `mutekiya_ramen_a1b2.md`).

## 2. The Canonical Markdown Template

Every memory written to disk by the Core Processing Engine must conform exactly to this layout structure.

```markdown
---
id: <UUIDv4>
type: <Enum: place | media | note | person>
category: qmd://<semantic/path/hierarchy>
date_saved: <ISO_8601_Date>
source: <string>
tags: [<string>, <string>]
<...Optional Plugin Keys...>
---

# <Extracted Title / Entity Name>

## Distilled Memory Items
- **<Key Point 1>**
- **<Key Point 2>**

---
## Raw Source
<The unmodified original text or parsed HTML body>
```

## 3. Zod Schemas

This is the exact Zod schema the `an-export` and Core Engine must use for extraction and formatting. 

### 3.1 Base Frontmatter Schema

```typescript
import { z } from 'zod';

export const MemoryTypeEnum = z.enum(["place", "media", "note", "person"]);

export const BaseFrontmatterSchema = z.object({
  /** A stable UUIDv4 unique to this discrete memory */
  id: z.string().uuid(),
  
  /** The structural type for directory routing */
  type: MemoryTypeEnum,
  
  /** A QMD Context URI (e.g. 'qmd://travel/food/japan') */
  category: z.string().startsWith('qmd://'),
  
  /** The ISO string date when this memory was initially saved/bookmarked */
  date_saved: z.string().datetime(),
  
  /** The system that originally captured this (e.g. 'apple_notes', 'x_bookmark') */
  source: z.string(),
  
  /** 1-5 core descriptive tags. Never more than 5. */
  tags: z.array(z.string()).max(5),

  /** The original URL if applicable (e.g. Safari Bookmark, Reddit thread) */
  url: z.string().url().optional(),
});

export type BaseFrontmatter = z.infer<typeof BaseFrontmatterSchema>;
```

### 3.2 Extracted Body Schema

When the LLM runs the extraction pipeline, we enforce strict structuring for the body text generation.

```typescript
export const MemoryExtractionSchema = z.object({
  title: z.string().describe("A concise, declarative title or entity name for the memory."),
  
  distilled_items: z.array(z.string()).min(1).max(7).describe(
    "1 to 7 atomic facts, quotes, or instructions extracted from the raw source. Must be standalone sentences."
  ),
  
  qmd_category: z.string().describe(
    "A hierarchical classification path starting with qmd://, e.g. qmd://tech/programming/python"
  ),
  
  type: MemoryTypeEnum,
  
  tags: z.array(z.string()).max(5).describe(
    "Between 1 and 5 thematic tags. lowercase, kebab-case."
  )
});
```

## 4. QMD Context URIs & Taxonomies

The `category` frontmatter string (`qmd://...`) is critical. QMD uses this path to build its semantic "Context Tree".

**Rule:** The Core Engine LLM must adhere to standard top-level domains rather than hallucinating infinite root paths.

**Allowed Top-Level Semantic Roots:**
1.  `qmd://tech/` (Programming, hardware, software tools, frameworks)
2.  `qmd://travel/` (Geography, restaurants, itineraries, tourism)
3.  `qmd://health/` (Fitness plans, medical history, recipes, nutrition)
4.  `qmd://finance/` (Receipts, investing, budgeting, taxes)
5.  `qmd://media/` (Books to read, movie lists, music, games)
6.  `qmd://personal/` (Diary entries, relationships, goals, reflections)
7.  `qmd://admin/` (Household, manuals, bureaucracy)

**Path generation:** The LLM can generate 1 or 2 sub-paths under these roots. (e.g. `qmd://tech/react/server-components`).

## 5. Reference LLM Extraction Prompt

The `packages/llm-extractor` module must use a system prompt that enforces structured output. Below is the reference prompt:

```
You are a memory extraction engine for Kore, a personal knowledge management system.

Given raw text from a user's saved content, extract structured metadata as JSON.

## Rules
- Extract a concise, declarative title (not a sentence, but a name/label).
- Extract 1-7 atomic facts as standalone sentences. Each fact must be independently useful.
- Classify into exactly one QMD category path. Use ONLY these top-level roots:
  - qmd://tech/ (programming, hardware, software, frameworks)
  - qmd://travel/ (geography, restaurants, itineraries, tourism)
  - qmd://health/ (fitness, medical, recipes, nutrition)
  - qmd://finance/ (receipts, investing, budgeting, taxes)
  - qmd://media/ (books, movies, music, games)
  - qmd://personal/ (diary, relationships, goals, reflections)
  - qmd://admin/ (household, manuals, bureaucracy)
  You may add 1-2 sub-paths (e.g. qmd://travel/food/japan).
- Assign a type: "place", "media", "note", or "person".
- Generate 1-5 lowercase kebab-case tags.

## Example

Input: "John recommended Mutekiya in Ikebukuro for solo dining. Cash only, get the tsukemen. Usually a 30 min wait."

Output:
{
  "title": "Mutekiya Ramen",
  "distilled_items": [
    "Mutekiya is a ramen shop in Ikebukuro, Tokyo.",
    "Recommended by John for solo dining.",
    "Cash only.",
    "The tsukemen is the recommended order.",
    "Expect approximately a 30 minute wait."
  ],
  "qmd_category": "qmd://travel/food/japan",
  "type": "place",
  "tags": ["ramen", "ikebukuro", "solo-dining", "cash-only"]
}
```

## 6. Plugin Extended Types

When plugins (like `spatialite`) inject data via the `onIngestEnrichment` hook, they append fields directly to the Frontmatter. These fields are not tracked by the core schema validation, but QMD will still see them.

**Example: Spatial Plugin Addition**
```yaml
coordinates: [35.7289, 139.7115] # Inserted by kore-plugin-spatialite
location_name: "Ikebukuro, Tokyo"
```
